# fonts-source/

Pre-subset font sources for the build pipeline. **Outside `public/`**
on purpose — these multi-MB files would otherwise ship to clients,
defeating the whole subset.

## Files

- `NotoSerifSC-{400,500}.woff2` (~1 MB each) — full CJK Unified
  Ideographs block. Source of truth for the body-text subset that
  lands in `public/fonts/NotoSerifSC-{400,500}.woff2`. Sourced from
  the spec's font drop; identical to what was in `public/fonts/`
  before the subset script existed.

- `NotoSerifSC-{400,500}-og.ttf` (~400 KB each) — uncompressed TTF
  copies covering body chars + every post-title char + ASCII. Read
  at build time by `src/pages/og/[...slug].png.ts` (satori needs
  TTF, not woff2). Regenerated by the subset script alongside the
  woff2 outputs.

## Regenerating

```sh
npm run subset-fonts
# === python3 scripts/subset-noto-serif-sc.py ===
```

Re-run when:

- a new post introduces CJK characters not in the current subset
  (visible as "豆腐块" / tofu boxes in the body), **or**
- a new post title introduces characters that satori can't render
  (visible as missing glyphs in the OG image).

The script reads from this directory, never overwrites the woff2
sources here, and writes to `public/fonts/` (body subset) and back
into this directory (OG TTFs).

## Don't

- Don't add the 400/500 source woff2 to `public/fonts/raw/` — that
  was the previous layout, now removed.
- Don't track the OG TTFs as the canonical source — they're a
  build artifact, regenerated from the woff2.
